## Overview

ChatOps allows you to manage feature flags and other operations directly from GitHub issue and pull request comments using slash commands.

## Getting Started

ChatOps is powered by a GitHub Actions workflow that responds to comments starting with `/chatops`.

### Prerequisites

- GitHub repository with Actions enabled
- Write permissions for the GitHub token
- Feature flags configuration file (`feature-flags.json`)

## Available Commands

### Feature Flag Commands

#### Enable a Feature

Enable a feature flag by setting it to `true`:

```
/chatops run feature set <flag_name>
```

**Example:**

```
/chatops run feature set new-dashboard
```

**Result:**
- Sets the flag to `enabled: true`
- Updates the `updatedAt` timestamp
- Commits the change to the repository
- Posts a confirmation comment

#### Disable a Feature

Disable a feature flag by setting it to `false`:

```
/chatops run feature unset <flag_name>
```

**Example:**

```
/chatops run feature unset beta-features
```

#### List All Flags

View all feature flags and their current status:

```
/chatops run feature list
```

**Example output:**

```
📋 Current Feature Flags:

✅ new-dashboard: enabled (updated: 2024-01-15T10:30:00.000Z)
❌ beta-features: disabled (updated: 2024-01-10T08:00:00.000Z)
✅ caching: enabled (updated: 2024-01-12T14:20:00.000Z)
```

## Workflow Configuration

The ChatOps workflow is defined in `.github/workflows/chatops.yml`:

```yaml
name: ChatOps

on:
  issue_comment:
    types: [created]

jobs:
  chatops:
    if: startsWith(github.event.comment.body, '/chatops')
    runs-on: ubuntu-latest
    permissions:
      contents: write
      issues: write
      pull-requests: write
    
    steps:
      - name: Checkout
        uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Parse and execute command
        run: |
          # Command parsing and execution logic
          # See full workflow for details
      
      - name: Commit changes
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add feature-flags.json
          git commit -m "Update feature flags via ChatOps"
          git push
      
      - name: Comment result
        run: |
          # Post result as comment
```

## Usage Examples

### Gradual Feature Rollout

1. **Development Phase**: Enable feature for testing
   ```
   /chatops run feature set new-algorithm
   ```

2. **Monitor**: Watch metrics and logs

3. **Rollback if needed**: Quick disable without deployment
   ```
   /chatops run feature unset new-algorithm
   ```

4. **Production**: Keep enabled and remove flag from code later

### Emergency Kill Switch

When something goes wrong in production:

```
/chatops run feature unset problematic-feature
```

Changes take effect immediately without requiring a deployment.

### A/B Testing

Enable features for specific test groups:

```
# Enable experiment
/chatops run feature set experiment-v2

# Check results, then disable
/chatops run feature unset experiment-v2
```

## Permissions

The ChatOps workflow requires specific GitHub token permissions:

- **contents: write** - To commit feature flag changes
- **issues: write** - To post comments on issues
- **pull-requests: write** - To post comments on PRs

These are configured in the workflow file under `permissions`.

## Security Considerations

<Warning>
ChatOps commands can modify your repository. Ensure proper access controls:

- Only repository collaborators can trigger workflows
- Review the workflow logs for all changes
- Consider requiring approval for sensitive operations
- Use branch protection rules for critical files
</Warning>

## Troubleshooting

### Command not executing

**Problem**: You posted a ChatOps command but nothing happened.

**Solutions**:
- Ensure the comment starts exactly with `/chatops` (case-sensitive)
- Check that GitHub Actions is enabled for your repository
- Verify you have the necessary permissions
- Review the Actions tab for error logs

### Permission denied errors

**Problem**: Workflow fails with permission errors.

**Solutions**:
- Check workflow permissions in `.github/workflows/chatops.yml`
- Ensure the GitHub token has write access
- Verify branch protection rules aren't blocking commits

### Command not found

**Problem**: ChatOps responds that the command doesn't exist.

**Solutions**:
- Check command syntax (e.g., `feature set` not `set feature`)
- Ensure you're using a supported command
- Review the workflow file for available commands

## Extending ChatOps

You can add custom commands by modifying the workflow file:

```yaml
- name: Custom command
  if: contains(github.event.comment.body, '/chatops run custom')
  run: |
    # Your custom logic here
    echo "Executing custom command"
```

## Example Workflow

Here's a complete example of using ChatOps for a feature rollout:

1. **Create feature flag**:
   ```javascript
   // In your code
   if (featureFlags.isEnabled('new-ui')) {
     renderNewUI();
   } else {
     renderOldUI();
   }
   ```

2. **Initially disable**:
   ```
   /chatops run feature unset new-ui
   ```

3. **Test in staging**:
   ```
   /chatops run feature set new-ui
   ```

4. **Monitor metrics** in your observability platform

5. **Production rollout**:
   ```
   /chatops run feature set new-ui
   ```

6. **Check status**:
   ```
   /chatops run feature list
   ```

7. **If issues arise**:
   ```
   /chatops run feature unset new-ui
   ```
