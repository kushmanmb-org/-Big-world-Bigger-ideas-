# ISO/IEC 27001:2013 Certification Fetcher

A comprehensive module for managing and tracking ISO/IEC 27001:2013 Information Security Management System (ISMS) certification information for kushmanmb.

## Overview

ISO/IEC 27001:2013 is an international standard that specifies requirements for establishing, implementing, maintaining, and continually improving an information security management system (ISMS). This module provides tools to track compliance status, manage security controls, and generate comprehensive reports.

## Features

- 📋 **Comprehensive Certification Tracking**: Full ISO 27001:2013 standard coverage
- 🔒 **114 Security Controls**: Track all Annex A controls across 14 domains
- 📊 **Compliance Reporting**: Generate detailed compliance reports and status summaries
- ✅ **Repository-Specific Compliance**: Track security measures specific to this blockchain repository
- 💾 **Caching**: Automatic caching with configurable timeout (default: 1 hour)
- 📈 **Progress Tracking**: Monitor implementation progress across all control domains
- 🎯 **Recommendations**: Get actionable recommendations based on compliance status

## Installation

The module is part of the repository and requires no additional installation beyond the standard dependencies:

```bash
npm install
```

## Quick Start

```javascript
const ISO27001Fetcher = require('./src/iso27001.js');

// Create a fetcher instance for kushmanmb
const fetcher = new ISO27001Fetcher('kushmanmb');

// Get full certification information
const info = fetcher.getCertificationInfo();
console.log(fetcher.formatCertificationInfo(info));

// Get compliance status
const status = fetcher.getComplianceStatus();
console.log(`Compliance: ${status.percentage}%`);

// Generate full compliance report
const report = fetcher.generateComplianceReport();
console.log(report);
```

## ISO/IEC 27001:2013 Standard Structure

### Main Clauses

The standard consists of 7 main clauses (4-10):

1. **Clause 4: Context of the Organization** - Understanding the organization and its context, interested parties, and ISMS scope
2. **Clause 5: Leadership** - Leadership commitment, information security policy, and organizational roles
3. **Clause 6: Planning** - Risk assessment, risk treatment, and information security objectives
4. **Clause 7: Support** - Resources, competence, awareness, communication, and documented information
5. **Clause 8: Operation** - Operational planning and control, risk assessment and treatment
6. **Clause 9: Performance Evaluation** - Monitoring, measurement, analysis, evaluation, internal audit, and management review
7. **Clause 10: Improvement** - Nonconformity, corrective action, and continual improvement

### Annex A Controls

The standard includes 114 security controls organized into 14 domains:

- **A.5**: Information Security Policies (2 controls)
- **A.6**: Organization of Information Security (7 controls)
- **A.7**: Human Resource Security (6 controls)
- **A.8**: Asset Management (10 controls)
- **A.9**: Access Control (14 controls)
- **A.10**: Cryptography (2 controls)
- **A.11**: Physical and Environmental Security (15 controls)
- **A.12**: Operations Security (14 controls)
- **A.13**: Communications Security (7 controls)
- **A.14**: System Acquisition, Development and Maintenance (13 controls)
- **A.15**: Supplier Relationships (5 controls)
- **A.16**: Information Security Incident Management (7 controls)
- **A.17**: Information Security Aspects of Business Continuity (4 controls)
- **A.18**: Compliance (8 controls)

## API Reference

### Constructor

#### `new ISO27001Fetcher(owner)`

Creates a new ISO27001 certification fetcher instance.

**Parameters:**
- `owner` (string, required): The owner/organization name (e.g., 'kushmanmb')

**Returns:** ISO27001Fetcher instance

**Example:**
```javascript
const fetcher = new ISO27001Fetcher('kushmanmb');
```

### Methods

#### `getCertificationInfo()`

Get comprehensive ISO 27001:2013 certification information including standard details, clauses, controls, benefits, and compliance areas.

**Returns:** Object containing:
- `standard`: Standard name (ISO/IEC 27001:2013)
- `standardName`: Full name of the standard
- `version`: Standard version (2013)
- `owner`: Owner/organization name
- `status`: Current certification status
- `description`: Standard description
- `scope`: Array of security management scope areas
- `clauses`: Array of standard clauses (4-10)
- `controls`: Object with control domains and implementation status
- `benefits`: Array of certification benefits
- `complianceAreas`: Repository-specific compliance areas
- `lastUpdated`: ISO date string of last update
- `nextReview`: ISO date string for next review

**Example:**
```javascript
const info = fetcher.getCertificationInfo();
console.log(`Standard: ${info.standard}`);
console.log(`Total Controls: ${info.controls.totalControls}`);
```

#### `getComplianceStatus()`

Get a summary of the current compliance status with percentages and overall assessment.

**Returns:** Object containing:
- `owner`: Owner name
- `totalControls`: Total number of controls
- `implementedControls`: Number of implemented controls
- `percentage`: Compliance percentage (0-100)
- `status`: Overall status ('Good' or 'In Progress')
- `lastAssessment`: ISO date string of last assessment

**Example:**
```javascript
const status = fetcher.getComplianceStatus();
console.log(`Compliance: ${status.percentage}%`);
console.log(`Status: ${status.status}`);
```

#### `getControlStatus()`

Get detailed control implementation status by domain.

**Returns:** Object containing:
- `owner`: Owner name
- `standard`: Standard name
- `totalControls`: Total number of controls
- `domains`: Array of control domains with implementation counts
- `summary`: Object with counts of fully/partially/not implemented domains

**Example:**
```javascript
const controlStatus = fetcher.getControlStatus();
controlStatus.domains.forEach(domain => {
  const percentage = Math.round((domain.implemented / domain.controls) * 100);
  console.log(`${domain.domain}: ${percentage}%`);
});
```

#### `formatCertificationInfo([info])`

Format certification information as a readable string.

**Parameters:**
- `info` (Object, optional): Certification info object. If not provided, calls `getCertificationInfo()`

**Returns:** Formatted string with certification details

**Example:**
```javascript
const formatted = fetcher.formatCertificationInfo();
console.log(formatted);
```

#### `generateComplianceReport()`

Generate a comprehensive compliance report with executive summary, control domain status, repository compliance areas, and recommendations.

**Returns:** Formatted compliance report string

**Example:**
```javascript
const report = fetcher.generateComplianceReport();
console.log(report);
```

#### `clearCache()`

Clear all cached data.

**Example:**
```javascript
fetcher.clearCache();
```

#### `getCacheStats()`

Get cache statistics including size, timeout, and cached entries.

**Returns:** Object containing:
- `size`: Number of cached entries
- `timeout`: Cache timeout in milliseconds
- `entries`: Array of cached keys

**Example:**
```javascript
const stats = fetcher.getCacheStats();
console.log(`Cache size: ${stats.size} entries`);
```
