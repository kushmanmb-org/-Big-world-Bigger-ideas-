## Prerequisites

Before installing Big World Bigger Ideas, ensure you have:

- **Node.js**: Version 14.0.0 or higher
- **npm**: Usually comes with Node.js
- A code editor (VS Code, Sublime Text, etc.)

## Installation Methods

### NPM (Recommended)

```bash
npm install big-world-bigger-ideas
```

### Yarn

```bash
yarn add big-world-bigger-ideas
```

### PNPM

```bash
pnpm add big-world-bigger-ideas
```

## Verify Installation

After installation, verify that the package is installed correctly:

```javascript
const bigWorld = require('big-world-bigger-ideas');
console.log('Package loaded successfully!');
```

## Module System

This package uses **CommonJS** modules (`require`/`module.exports`), not ES modules.

### Importing the Entire Package

```javascript
const bigWorld = require('big-world-bigger-ideas');
```

### Importing Specific Modules

```javascript
const { 
  ERC721Fetcher, 
  ERC20Fetcher,
  BitcoinMiningFetcher,
  BlockchairFetcher,
  TokenUUID,
  FeatureFlags
} = require('big-world-bigger-ideas');
```

## Configuration

### Environment Variables

Some modules may require environment variables. Create a `.env` file in your project root:

```bash
# Example .env file
ETHEREUM_RPC_URL=https://ethereum.publicnode.com
BLOCKCHAIR_API_KEY=your_api_key_here
ETHERSCAN_API_KEY=your_api_key_here
```

<Note>
Never commit your `.env` file to version control. Add it to `.gitignore`.
</Note>

### Feature Flags

If you're using the feature flags module, create a `feature-flags.json` file:

```json
{
  "flags": {
    "new-feature": {
      "enabled": true,
      "updatedAt": "2024-01-01T00:00:00.000Z"
    }
  }
}
```

## Common Issues

<AccordionGroup>
  <Accordion title="Module not found error">
    **Problem**: `Cannot find module 'big-world-bigger-ideas'`
    
    **Solution**: 
    - Ensure you've run `npm install big-world-bigger-ideas`
    - Check that `node_modules` directory exists
    - Try deleting `node_modules` and running `npm install` again
  </Accordion>

  <Accordion title="Node version error">
    **Problem**: Package requires Node.js >= 14.0.0
    
    **Solution**:
    - Check your Node.js version: `node --version`
    - Upgrade Node.js if needed: https://nodejs.org/
    - Consider using nvm (Node Version Manager) for managing Node.js versions
  </Accordion>

  <Accordion title="CommonJS vs ES Modules">
    **Problem**: `require is not defined` or import/export errors
    
    **Solution**:
    - This package uses CommonJS, not ES modules
    - Use `require()` instead of `import`
    - If you need ES modules, consider using dynamic imports
  </Accordion>

  <Accordion title="API rate limits">
    **Problem**: Getting rate limited by blockchain APIs
    
    **Solution**:
    - Implement caching in your application
    - Use API keys where available (Etherscan, Blockchair)
    - Consider running your own RPC node
  </Accordion>
</AccordionGroup>
